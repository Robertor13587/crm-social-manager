4. Autenticazione JWT

Flusso richiesto:

Frontend → POST /auth/login → Backend genera JWT → 
Frontend memorizza → tutte le richieste successive includono:
Authorization: Bearer {token}

Validazione obbligatoria per TRAE:

[ ] login() deve salvare JWT
[ ] api() wrapper deve inserire Authorization
[ ] intercettare 401 → forzare logout
[ ] token NON va mai inviato via querystring

5. Wrapper API lato frontend (da implementare)

const API_BASE = "https://socialcrm.robdev.website";

async function api(path, options = {}) {
  const token = localStorage.getItem("jwt_token");

  const res = await fetch(API_BASE + path, {
    ...options,
    headers: {
      "Content-Type": "application/json",
      "Authorization": "Bearer " + token,
      ...(options.headers || {})
    }
  });

  if (res.status === 401) {
    localStorage.removeItem("jwt_token");
    location.href = "/login";
  }

  return res.json();
}

10. Controlli su Nginx (per conferma)

Configurazione richiesta:

server {
    server_name socialcrm.robdev.website;

    listen 443 ssl;
    ssl_certificate     /etc/letsencrypt/live/socialcrm.robdev.website/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/socialcrm.robdev.website/privkey.pem;

    location / {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
    }
}
